# 🔺 Apex Legends Packs Calculator - Frontend Project Plan (2026)

เอกสารแผนงานและโครงสร้างสถาปัตยกรรมสำหรับระบบคำนวณกล่องสุ่มเกม Apex Legends เพื่อหาประวัติการเปิดเข้าใกล้จุดการันตี Heirloom Shards (500 กล่อง) โดยเน้นสถาปัตยกรรมแบบ **Client-side 100%** ประมวลผลและบันทึกข้อมูลบนเบราว์เซอร์ของผู้ใช้โดยตรง ไม่ต้องมีระบบหลังบ้าน (Backend) และฐานข้อมูล

---

## 📋 1. ภาพรวมระบบ (System Overview)
เว็บแอปพลิเคชันรูปแบบ Single Page Application (SPA) ที่ช่วยให้ผู้ใช้สามารถคำนวณจำนวน Apex Packs ได้แบบ Real-time และมีระบบ **Auto-Save บันทึกข้อมูลฝังไว้ในเครื่องผู้ใช้โดยอัตโนมัติ** ทำให้เมื่อปิดเว็บแล้วเปิดใหม่ ข้อมูลเดิมก็ยังคงอยู่ ช่วยลดค่าใช้จ่ายในการเช่าเซิร์ฟเวอร์หลังบ้าน และมอบประสบการณ์การใช้งานที่รวดเร็ว ไม่มีอาการหน่วงจากการรอโหลด API

### แนวคิดการปรับปรุง UI/UX ให้ใช้งานง่ายกว่าเว็บเดิมในท้องตลาด:
* **Progressive Disclosure:** แยกการแสดงผลตามเงื่อนไขยุคเก่า (Single Season) และยุคใหม่ (2 Splits) อย่างสิ้นเชิง เพื่อลดความหนาแน่นของฟอร์มในหน้าจอเดียว
* **Batch Operations (Quick Actions):** เพิ่มปุ่มลัดช่วยอำนวยความสะดวก เช่น "ซื้อพาสทุกซีซั่น" เพื่อลดจำนวนการคลิกของผู้ใช้
* **Visual Gamification:** แสดงหลอดพลัง (Progress Bar) วิ่งเข้าใกล้เลเวล 500 เพื่อดึงดูดสายตาและทำให้เห็นภาพชัดเจนขึ้นว่าขาดอีกกี่กล่องจะการันตี Heirloom

## ⚙️ 2. เงื่อนไขและเกณฑ์การคำนวณ (Business Logic)

### กิจกรรมหลักที่ไม่ขึ้นกับซีซั่น
1. **Player Level (สูงสุดเลเวล 2000 ในปี 2026):**
   * เลเวล 2 - 20: ได้ 1 กล่องต่อเลเวล (รวม 19 กล่อง)
   * เลเวล 22 - 300: ได้ 1 กล่องทุกๆ 2 เลเวล (รวม 140 กล่อง)
   * เลเวล 305 - 500: ได้ 1 กล่องทุกๆ 5 เลเวล (รวม 40 กล่อง)
   * เลเวล 501 - 2000 (ระบบ Prestige 3 รอบ): ได้ 1 กล่องทุกๆ 10 เลเวล (รวมสูงสุด 150 กล่อง)
   * *หมายเหตุ: หากผู้เล่นเลเวลเต็ม 2000 จะได้กล่องสะสมจากเลเวลทั้งหมด 349 กล่อง*

2. **Miscellaneous Packs:** ช่องสำหรับให้ผู้ใช้พิมพ์ระบุจำนวนกล่องเพิ่มเติมเอง เช่น กล่องที่ได้รับจาก Event พิเศษ, โค้ดแจกฟรี, หรือสิทธิพิเศษจาก Twitch Prime Gaming

### กิจกรรมที่ขึ้นตรงกับประวัติซีซั่น
* **ยุคเก่า (Season 1 ถึง 21):** ระบบ 1 ซีซั่นมี 1 พาส
  * สายฟรี (Free Track) = **+7 กล่อง**
  * สายเปย์ (Premium Track) = **+8 กล่อง** (แบ่งเป็น 7 กล่องธรรมดา + 1 กล่องทองระดับ Legendary)
  * เก็บกระเป๋าแดงครบ (Treasure Packs) = **+1 กล่อง**
* **ยุคใหม่ (Season 22 ถึง ปัจจุบันปี 2026):** ระบบ 2 Splits ต่อ 1 ซีซั่น (ไม่มี Treasure Packs แล้ว)
  * **Split 1:** สายฟรี = **+7 กล่อง** | ซื้อ Premium = **+7 กล่อง** | ซื้อ Ultimate+ = **+14 กล่อง**
  * **Split 2:** สายฟรี = **+7 กล่อง** | ซื้อ Premium = **+7 กล่อง** | ซื้อ Ultimate+ = **+14 กล่อง**

## 💾 3. กลไกการจัดการข้อมูลหน้าบ้าน (State & Storage Strategy)
เนื่องจากระบบไม่มีฐานข้อมูลหลังบ้าน เราจะใช้ประโยชน์จาก **Web Storage API** ในการจัดการและรักษาข้อมูลของผู้ใช้งาน:

### 1. บันทึกข้อมูลผ่าน `localStorage`
ทุกครั้งที่ผู้ใช้งานคลิกเลือกสวิตช์ หรือพิมพ์เลเวล ระบบจะทำการแปลงโครงสร้างข้อมูล Object (State) ให้อยู่ในรูปแบบ JSON String แล้วนำไปบันทึกไว้ใน `localStorage` ของเบราว์เซอร์ภายใต้คีย์ที่ชื่อ `apex_tracker_data` ค้างไว้ตลอดกาลจนกว่าผู้ใช้จะกดล้างแคชหรือกดปุ่มรีเซ็ตบนหน้าเว็บ

### 2. โครงสร้างข้อมูล JSON สำหรับจัดเก็บ (Data Structure)
```json
{
  "playerLevel": 750,
  "miscPacks": 15,
  "seasonsData": {
    "1": { "isPremium": true, "completedTreasure": true },
    "21": { "isPremium": false, "completedTreasure": false },
    "22": { "s1Premium": true, "s1UltimatePlus": false, "s2Premium": false, "s2UltimatePlus": false },
    "29": { "s1Premium": false, "s1UltimatePlus": false, "s2Premium": false, "s2UltimatePlus": false }
  }
}
